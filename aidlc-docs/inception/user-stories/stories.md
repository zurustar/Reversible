# User Stories — Reversible

- **分割アプローチ**: ハイブリッド(Feature単位の骨格 + 主要ジャーニー横断ストーリー)
- **受け入れ基準**: Given-When-Then(GWT)
- **粒度**: 中(1機能=1〜2ストーリー)
- **範囲**: MVP中心 + 将来拡張は `[将来]` タグ
- **ペルソナ**: ケンジ(宅録トラックメイカー)/ アヤ(ライブ・ジャム志向)。詳細は [personas.md](personas.md)
- **凡例**: `[MVP]` = 初回リリース対象 / `[将来]` = 後続拡張 / FR/NFR/SEC/PBT は [requirements.md](../requirements/requirements.md) への参照

各ストーリーは INVEST(Independent, Negotiable, Valuable, Estimable, Small, Testable)を意識して記述。

---

## Feature: シーケンサー & トランスポート

### US-01 [MVP] シーケンサーの再生/停止 — FR-4.3
**As** アヤ(ライブ)、**I want** ワンアクションで再生と停止ができる、**so that** 演奏やチェックをすぐ始め/止められる。
- **GWT-1**: Given パターンが打ち込まれている, When 再生を押す, Then パターンが先頭ステップからループ再生される。
- **GWT-2**: Given 再生中, When 停止を押す, Then 再生が止まり、次の再生は先頭から始まる。
- **GWT-3**: Given 再生中, When 再度再生操作をする, Then 二重再生や音の重なりが起きない(状態は再生/停止の2値で一貫)。
- **ペルソナ**: アヤ ◎ / ケンジ ○

### US-02 [MVP] テンポ(BPM)の設定 — FR-4.3
**As** ケンジ、**I want** BPMを設定できる、**so that** 作りたい曲のテンポに合わせられる。
- **GWT-1**: Given 任意のBPM(例: 20–300の範囲内), When BPMを変更する, Then 再生速度がそのBPMに追従する。
- **GWT-2**: Given 再生中にBPMを変更, When 値を変える, Then 再生を止めずにテンポが滑らかに変わり、グリッチが起きない(NFR-1)。
- **GWT-3**: Given 範囲外の値を入力, When 確定しようとする, Then 範囲内にクランプ/拒否され、クラッシュしない。
- **ペルソナ**: ケンジ ○ / アヤ ◎

### US-03 [MVP] ドラムの16ステップ打ち込み — FR-4.4 / FR-3
**As** ケンジ、**I want** ドラム各音色の16ステップをオン/オフできる、**so that** ビートを組める。
- **GWT-1**: Given ドラム音色の1トラック, When 任意ステップをトグルする, Then そのステップの発音が切り替わり、UI上の状態も一致する。
- **GWT-2**: Given 打ち込んだパターンを再生, When 再生する, Then オンにしたステップのタイミングでのみ発音する。
- **ペルソナ**: ケンジ ◎ / アヤ ○

### US-04 [MVP] basslineベースラインの16ステップ打ち込み(音程) — FR-4.4 / FR-2
**As** ケンジ、**I want** 各ステップに音程(またはレスト)を割り当てられる、**so that** アシッドなベースラインを作れる。
- **GWT-1**: Given basslineトラック, When ステップに音程を設定する, Then 再生時にその音程で発音する。
- **GWT-2**: Given ステップをレスト(無音)に設定, When 再生する, Then そのステップでは発音しない。
- **ペルソナ**: ケンジ ◎

### US-05 [MVP] basslineステップのアクセント/スライド — FR-2.5
**As** ケンジ、**I want** ステップごとにアクセントとスライド(グライド)を設定できる、**so that** basslineらしい抑揚とうねりを付けられる。
- **GWT-1**: Given あるステップ, When アクセントをオンにする, Then そのステップの音量/フィルターの効きが強調される。
- **GWT-2**: Given 連続する2ステップ, When 前ステップにスライドを設定, Then 音程が次ステップへグライド(レガート)して繋がる。
- **ペルソナ**: ケンジ ◎ / アヤ ○

### US-06 [実装済み] シャッフル/スウィング — FR-4.1
**As** ケンジ、**I want** スウィングを調整できる、**so that** ノリ(グルーヴ)を出せる。
- **GWT-1**: Given スウィング量を設定, When 再生, Then 偶数/奇数ステップのタイミングがずれてハネる。
- **GWT-2(不変条件)**: Given 任意のスウィング量, When 適用, Then 1小節あたりの発音ステップ数は変わらない(PBT-03 候補)。
- **ペルソナ**: ケンジ ○ / アヤ ○

### US-07 [実装済み] パターンチェーン/ソングモード — FR-4.1
**As** アヤ、**I want** 複数パターンを並べて曲として再生できる、**so that** 展開のある演奏/楽曲にできる。
- **GWT-1**: Given 複数パターン, When 並び順を指定して再生, Then その順序で切れ目なく再生される。
- **ペルソナ**: アヤ ◎ / ケンジ ○

## Feature: bassline スタイル ベースライン・シンセ(音作り)

### US-08 [MVP] 波形選択(saw/square) — FR-2.2
**As** ケンジ、**I want** ノコギリ波と矩形波を切り替えられる、**so that** 基本の音色キャラを選べる。
- **GWT-1**: Given bassline, When 波形をsaw/squareで切り替える, Then 発音の音色が対応して変わる。
- **ペルソナ**: ケンジ ◎

### US-09 [MVP] basslineフィルターのリアルタイム操作 — FR-2.3 / NFR-1
**As** アヤ、**I want** Cutoff/Resonance/EnvMod/Decay/Accent を再生中に回せる、**so that** ライブでうねりを作れる。
- **GWT-1**: Given 再生中, When Cutoffを動かす, Then 音のブライトさがリアルタイムに変化し、音切れが起きない(NFR-1)。
- **GWT-2**: Given Resonanceを上げる, When 発音, Then フィルターの共鳴(ピーク)が強まり、bassline特有のうねりが出る。
- **GWT-3(範囲不変)**: Given 各ノブ [0,1], When 内部パラメータへ変換, Then 変換結果は定義域内に収まる(PBT-03 候補)。
- **ペルソナ**: アヤ ◎ / ケンジ ○

### US-10 [MVP] basslineのチューニングと音量 — FR-2.4
**As** ケンジ、**I want** Tuneとレベルを調整できる、**so that** 他の音とのピッチ/バランスを合わせられる。
- **GWT-1**: Given Tuneを変更, When 発音, Then 全体のピッチが上下する。
- **GWT-2**: Given レベルを変更, When 発音, Then 出力音量が対応して変わる。
- **ペルソナ**: ケンジ ◎

### US-13 [実装済み] 2台目のbassline — FR-2.6
**As** ケンジ、**I want** basslineをもう1台追加できる、**so that** 2本のベースラインを重ねられる。
- **GWT-1**: Given 2台目を追加, When それぞれに別パターン/音色を設定, Then 独立して同時に鳴る。
- **ペルソナ**: ケンジ ○

## Feature: ドラムマシン(音作り)

### US-11 [MVP] ドラム各音色のパラメータ/音量調整 — FR-3.3
**As** ケンジ、**I want** 各ドラム音色のTone/Decay/Tuning/音量を調整できる、**so that** キックやハットの質感を追い込める。
- **GWT-1**: Given ある音色, When Decayを変更, Then 発音の減衰時間が変わる。
- **GWT-2**: Given ある音色の音量, When 変更, Then ミックス中のその音色の音量が変わる。
- **ペルソナ**: ケンジ ◎ / アヤ ○

### US-12 [実装済み] 2台目のドラムマシン(analog+digital構成)/音色拡張 — FR-3.4
**As** ケンジ、**I want** もう1台のドラムマシンを使える、**so that** analog系とdigital系を同時に使える。
- **GWT-1**: Given 2台目を追加, When 打ち込み, Then 2台が独立して同時に鳴る。
- **ペルソナ**: ケンジ ○

## Feature: 保存・復元(ブラウザ内)

### US-14 [MVP] ブラウザへの保存と再読み込み時の復元 — FR-5.1 / FR-5.2
**As** ケンジ、**I want** 作業内容がブラウザに保存され、開き直すと復元される、**so that** 作業を失わない。
- **GWT-1**: Given パターンを編集, When 保存(または自動保存)する, Then localStorage等に状態が保持される。
- **GWT-2**: Given 保存済み, When ページを再読み込みする, Then 直前の状態が完全に復元される(ラウンドトリップ, PBT-02 候補)。
- **ペルソナ**: ケンジ ◎ / アヤ ○

## Feature: エクスポート / インポート(JSON) — FR-5.4〜5.7

### US-15 [MVP] 曲をJSONでエクスポート(ダウンロード) — FR-5.4
**As** ケンジ、**I want** 曲データをJSONファイルとしてダウンロードできる、**so that** バックアップや共有ができる。
- **GWT-1**: Given 作成した曲, When エクスポートを実行, Then `song.json` 相当のファイルがブラウザからダウンロードされる(サーバ非経由)。
- **GWT-2**: Given エクスポートしたJSON, When 中身を確認, Then スキーマ/バージョン情報を含み、曲の全状態を表現している(FR-5.3)。
- **ペルソナ**: ケンジ ◎ / アヤ ○

### US-16 [MVP] JSONのコピペでインポート — FR-5.5
**As** アヤ、**I want** JSONテキストを貼り付けて読み込める、**so that** 事前に用意した曲を素早く復元して演奏に入れる。
- **GWT-1**: Given 妥当なJSONテキスト, When テキストエリアに貼り付けて読み込む, Then 曲状態が復元される。
- **GWT-2(ラウンドトリップ)**: Given US-15でエクスポートしたJSON, When それを貼り付けて読み込む, Then 元の曲状態が完全に一致して復元される(PBT-02, FR-5.7)。
- **ペルソナ**: アヤ ◎ / ケンジ ◎

### US-17 [MVP] .jsonファイルのアップロードでインポート — FR-5.5(任意)
**As** ケンジ、**I want** `.json`ファイルを選んで読み込める、**so that** コピペせずファイルから復元できる。
- **GWT-1**: Given 妥当な `.json` ファイル, When 選択して読み込む, Then 曲状態が復元される(コピペ経路と同じ結果)。
- **ペルソナ**: ケンジ ○
- **備考**: FR-5.5では任意。コピペ(US-16)が必須。

### US-18 [MVP] インポートの検証と安全なエラー処理 — FR-5.6 / SEC-05 / SEC-13 / SEC-15
**As** ケンジ、**I want** 壊れた/不正なデータを読み込んでもアプリが落ちない、**so that** 安心して他人のJSONも試せる。
- **GWT-1**: Given 不正なJSON(構文エラー), When 読み込む, Then 分かりやすいエラーが表示され、既存の作業状態は保持され、クラッシュしない(SEC-15 フェイルセーフ)。
- **GWT-2**: Given スキーマ不一致/型不正のデータ, When 読み込む, Then 検証で弾かれ、取り込まれない(SEC-05 入力検証)。
- **GWT-3**: Given 想定外の埋め込み(実行コード等を装う内容), When 読み込む, Then 安全にデータとしてのみ扱われる(SEC-13 安全なデシリアライズ)。
- **GWT-4**: Given 非対応バージョンのデータ, When 読み込む, Then バージョン差を検知し、可能なら移行、不可能なら明示エラー。
- **ペルソナ**: ケンジ ◎ / アヤ ○

## Feature: UI

### US-19 [MVP] 機能優先UIから全操作にアクセスできる — FR-6.1 / NFR-1
**As** アヤ、**I want** 主要コントロール(打ち込み/再生/ノブ)に素早く到達でき、操作が即応する、**so that** 制作でも本番でも迷わず触れる。
- **GWT-1**: Given アプリ画面, When 各コントロールを操作, Then 遅延なく反映され、UIとオーディオの状態がずれない。
- **GWT-2**: Given 主要機能(bassline/ドラム/シーケンサー/保存/入出力), When 画面を見る, Then 深い階層に隠れず到達できる。
- **ペルソナ**: アヤ ◎ / ケンジ ○
- **備考**: モダン/装飾UI(FR-6.2)は `[将来]`。

### US-20 [実装済み] エフェクトの挿入 — FR-7
**As** ケンジ、**I want** ディストーション/ディレイ/PCF/コンプ等を挿せる、**so that** 音を仕上げられる。
- **GWT-1**: Given エフェクトを有効化, When 発音, Then 出力にそのエフェクトがかかる。
- **ペルソナ**: ケンジ ○ / アヤ ○

## 横断ジャーニー(主要ユーザーフロー)

### JS-1 [MVP] 宅録: アシッドループを一気通貫で作る(ケンジ)
**As** ケンジ、**I want** 起動から保存・エクスポートまでを数分で完結できる、**so that** 思いつきを逃さずに形にできる。
- **GWT-1**: Given アプリを開いた直後, When ドラム(US-03)とbassline(US-04/US-05)を打ち込む, Then すぐにアシッドなループが鳴る。
- **GWT-2**: Given ループが鳴っている, When basslineフィルターを追い込み(US-09), Then 望むうねりが得られる。
- **GWT-3**: Given 満足した状態, When 保存(US-14)しエクスポート(US-15), Then 作業が保持され、JSONを持ち出せる。
- **関連**: US-01,02,03,04,05,09,14,15

### JS-2 [MVP] ライブ: 止めずに操作し続けても破綻しない(アヤ)
**As** アヤ、**I want** 再生を止めずにパラメータを操作しても音が破綻しない、**so that** 安心してライブで使える。
- **GWT-1**: Given 事前JSONを読み込み(US-16), When 再生開始(US-01), Then 用意したパターンが即鳴る。
- **GWT-2**: Given 再生継続中, When フィルターやBPM(US-09/US-02)を連続操作, Then 音切れ/グリッチなく滑らかに変化する(NFR-1)。
- **関連**: US-16,01,09,02
