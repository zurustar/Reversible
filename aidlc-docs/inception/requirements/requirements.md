# Requirements — Reversible

## Intent Analysis Summary
- **User Request**: ブラウザ上で動く、オリジナルのアシッド・グルーヴボックス(ベースライン・シンセ×2 + ドラムマシン×2 + ステップシーケンサー + エフェクト)を作りたい。
- **Request Type**: New Project (Greenfield)
- **Scope Estimate**: Multiple Components(音源エンジン、シーケンサー、UI、状態管理)
- **Complexity Estimate**: Moderate → Complex(リアルタイムDSP合成が中心。UI/シーケンサーは段階的に拡張)
- **Request Clarity**: Clear(質問への回答により確定)

## Product Vision
ブラウザ上で動く、オリジナルのアシッド系グルーヴボックス。ベースライン合成、ドラムマシン、16ステップシーケンサーを核とし、**知的財産はすべて独自**とする。名称・音源・グラフィックはすべて独自で、いかなるメーカーの商標・著作物・素材も一切使用しない。

## Functional Requirements

### FR-1: 実行プラットフォーム
- **FR-1.1** Webブラウザ上で動作する(インストール不要、静的サイトとして配信可能)。
- **FR-1.2** 実装は TypeScript、音声生成は Web Audio API を用いる。
- **FR-1.3** モダンな主要ブラウザ(Chrome / Firefox / Safari / Edge の最新版)で動作すること。
- **FR-1.4** ビルド成果物は **単一の自己完結HTMLファイル**として生成でき、サーバ不要で**ファイルを開くだけ(`file://`)で動作**すること(音は合成のみでランタイムのアセット取得が無い)。「ダウンロードして即実行」の配布形態を満たす。

### FR-2: Bassline スタイル ベースライン・シンセ(MVP: ×1)
- **FR-2.1** リアルタイム・アナログモデリング合成で音を生成する(サンプル不使用の独自実装)。
- **FR-2.2** 波形選択: ノコギリ波(saw)/ 矩形波(square)。
- **FR-2.3** レゾナント・ローパスフィルタ(bassline特有の「うねり」を出せること)。パラメータ: Cutoff, Resonance, Env Mod, Decay, Accent。
- **FR-2.4** チューニング(Tune)、音量(Volume / Level)。
- **FR-2.5** ステップごとの Accent(アクセント)と Slide(グライド/レガート)。
- **FR-2.6** [実装済み] basslineを**2台**(Bassline A / B)実装。`Pattern.bassline` は `BasslineTrack[]`(BASSLINE_COUNT=2)、各トラックに独立した音源(`bassline-0`/`bassline-1`)とUIセクション。
- **FR-2.7** [実装済み] 基本ノブ(Tune/Cutoff/Resonance/EnvMod/Decay/Accent/Volume/波形)に加え、拡張ノブとして **Drive**(オーバードライブ=波形整形による歪み)と **Slide**(スライドのグライド時間)を追加。acid 系のベースラインで一般的で musical。

### FR-3: ドラムマシン(MVP: ×1)
- **FR-3.1** リアルタイム・アナログモデリング合成でドラム音を生成する(サンプル不使用)。
- **FR-3.2** [実装済み] 11音色を合成で実装: Bass Drum / Snare / Low・Mid・Hi Tom / Rim Shot / Clap / Cowbell / Cymbal / Closed Hat / Open Hat。将来さらに追加可能(voiceId追加 + validator正規化で旧データも読める)。
- **FR-3.3** [実装済み] 各音色に Level に加え、analog drum machine風の固有パラメータ: BD=Tone/Decay、Snare=Tone/Snappy、Tom=Tune、Cymbal=Tone/Decay、CH/OH=Decay、Cowbell=Tune。合成に反映(Snappyはボディとノイズのバランス, Toneはアタック/明るさ 等)。
- **FR-3.4** [実装済み] ドラムマシンを**2台**(analog系 + digital系)実装。`Pattern.drums` は `DrumTrack[]`(DRUM_MACHINE_COUNT=2)、各台に独立音源(`drums-0`=analog / `drums-1`=digital、合成を style で切替)とUIセクション。

### FR-4: ステップシーケンサー
- **FR-4.1** 目標(最終形): 16ステップのパターン、パターンをつなげたソング(パターンチェーン)、basslineのアクセント/スライドのステップ入力、シャッフル/スウィング、ドラムのアクセント等、クラシック相当の機能。
- **FR-4.2** 段階的実装を許容: 構築工程の初期段階では、まず基本の16ステップ・パターン再生(パターンチェーンや高度機能なし)を先に作ってよい。最終的にFR-4.1の全機能を目指す。
- **FR-4.3** BPM(テンポ)設定と、再生 / 停止 / (可能なら)一時停止。
- **FR-4.4** 各トラック(bassline・ドラム各音色)に対するステップのオン/オフ入力。
- **FR-4.5** タイミングは Web Audio のオーディオクロックに同期し、UIスレッドのタイマーに依存しない安定した再生を行う(look-ahead スケジューリング)。

### FR-5: パターン/プロジェクトの保存・読み込み・インポート/エクスポート
- **FR-5.1** 作成したパターン(および将来のソング)をシリアライズして保存・復元できる(例: JSON、ブラウザの localStorage）。
- **FR-5.2** シリアライズ ⇄ デシリアライズはラウンドトリップで元の状態を完全に復元できること(PBT対象)。
- **FR-5.3** 保存フォーマットにはスキーマ/バージョン情報を持たせ、将来の拡張に備える。
- **FR-5.4 エクスポート(ダウンロード)**: 作成した曲データを **JSON** 形式でローカルにダウンロードできる。
  - ダウンロードはブラウザのファイル保存(Blob + ダウンロードリンク等)で行い、サーバを介さない。
  - ファイル名・拡張子は `.json` を付与する(例: `song.json`)。
  - 注: フォーマットは JSON のみ(実装容易性を優先。YAML等の追加は将来の拡張として任意)。
- **FR-5.5 インポート(読み込み)**: 外部の曲データ(JSON)を取り込んで復元できる。取り込み手段:
  - **テキストのコピー&ペースト入力**(必須): テキストエリアに JSON を貼り付けて読み込む。
  - **ファイルアップロード**(任意): `.json` ファイルを選択して読み込む。
- **FR-5.6 インポート時の検証**: 取り込むデータは信頼できない外部入力として扱い、スキーマ/型検証を行う(SECURITY-05 / SECURITY-13)。不正・破損・非対応バージョンのデータは、クラッシュせずに分かりやすいエラーとして扱う(SECURITY-15 / フェイルセーフ)。
- **FR-5.7 ラウンドトリップ整合**: エクスポートした曲データ(JSON)を再インポートすると、元の曲状態が完全に復元されること(PBT-02 対象)。

### FR-6: UI(段階的)
- **FR-6.1** MVP: 最小限・機能優先のUI(音のエンジンとシーケンサーの動作を確認・操作できること)。派手な装飾より、パラメータ操作とステップ入力の実用性を優先。
- **FR-6.2** [実装済み(初版)] ハードウェア風のリッチUIスキン(メタリックな筐体パネル、LED風ステップ、ロゴ、発光する点灯表示)を適用。CSSのみで実現し、UIとオーディオエンジンは疎結合のまま。見た目の微調整は継続。
- **FR-6.3** グラフィック/アイコン等の素材はすべてオリジナル。既存製品のスキンや画像は使用しない。

### FR-7: エフェクト(将来拡張)
- **FR-7.1** [実装済み] master バスに **Distortion / Delay / PCF(パターン/LFO変調フィルタ)/ Compressor** のエフェクトチェーンを実装(各 on/off + パラメータ、`effects.ts` の `FxChain`)。設定は Song モデル(`effects`)に保存・JSON入出力対応。

## Non-Functional Requirements

### NFR-1: パフォーマンス / リアルタイム性
- グリッチ(音切れ)のない安定したリアルタイム再生。オーディオコールバックをブロックしない。
- シーケンサーは look-ahead スケジューリングでタイミング精度を確保する。
- 一般的なノートPCで、bassline×1 + ドラム×1 + 16ステップ再生が余裕を持って動作すること。

### NFR-2: 移植性 / 配信
- 静的アセットのみで配信可能(サーバサイド処理に依存しない)。ビルド成果物をそのままホスティングできる。

### NFR-3: 保守性 / 拡張性
- オーディオエンジン、シーケンサー(音楽的ロジック)、UI、永続化を明確に分離する。
- basslineを2台・ドラムを2台へ、またエフェクトを追加できるよう、デバイスをプラガブルに扱える設計とする。

### NFR-4: テスト容易性
- 音楽的ロジック(シーケンサーのステップ計算、タイミング、シリアライズ、パラメータ変換、DSPの純粋関数部分)を副作用から分離し、テスト可能にする。
- Property-Based Testing(部分適用)の対象を設計時に特定する。

### NFR-5: セキュリティ(Security Baseline — 有効/Full)
- クライアントサイド静的Webアプリという性質上、多くのサーバ/クラウド向けルールは N/A。本プロジェクトで**適用対象**となる主なルール:
  - **SECURITY-04**: HTMLを配信するため、CSP等のHTTPセキュリティヘッダを設定する(配信時の設定として文書化)。
  - **SECURITY-05**: 外部入力(インポートするパターンファイル、URLパラメータ等)を検証する。
  - **SECURITY-10**: 依存関係のロックファイル、脆弱性スキャン、`latest` タグ不使用等のサプライチェーン管理。
  - **SECURITY-13**: 保存データ(パターンJSON)のデシリアライズを安全に行う(型検証/スキーマ検証、任意コード実行の回避)。外部CDN利用時はSRI。
  - **SECURITY-15**: 例外処理とフェイルセーフ(Web Audio初期化失敗、破損したパターン読み込み等でクラッシュせず安全に扱う)。
- サーバ、DB、認証、ネットワーク、IAM等に関するルール(SECURITY-01/02/03/06/07/08/09/11/12/14)は現時点の構成では **N/A**(バックエンド追加時に再評価)。

### NFR-6: レジリエンシー(Resiliency Baseline)
- **無効**(ユーザー選択B)。AWS Well-Architected由来の設計時レジリエンシー・ベースラインは本プロジェクトでは適用しない。

### NFR-7: プロパティベーステスト(PBT — 部分適用)
- 部分適用モード: **PBT-02(ラウンドトリップ)/ PBT-03(不変条件)/ PBT-07(ジェネレータ品質)/ PBT-08(シュリンク・再現性)/ PBT-09(フレームワーク選定)** を強制。その他は助言的(非ブロッキング)。
- 想定フレームワーク: TypeScript のため **fast-check**(NFR/Tech Stack 段階で確定)。
- 主なPBT対象候補:
  - パターン/曲のシリアライズ ⇄ デシリアライズ(ラウンドトリップ)。エクスポート → インポートで元状態が完全復元されること(JSON、FR-5.7)。
  - シーケンサーのステップ計算・不変条件(例: 16ステップは常に16、パターン長不変、スウィングでノート数不変)。
  - パラメータ変換(例: ノブ値 [0,1] ↔ 周波数/カットオフのマッピングが範囲内)。

## Assumptions & Constraints
- **A-1**: バックエンドサーバは持たない(MVPは純クライアント)。将来必要になれば別途評価。
- **A-2**: すべての音源はコードによる合成(サンプル素材は使わない)。ドラムも合成で生成。
- **A-3**: 名称・グラフィック・音源はすべてオリジナル。いかなるメーカー/製品の商標・著作物・型番も使用しない。ドキュメント内でも一般的な表現(「ベースライン・シンセ」「アナログ系/デジタル系ドラムマシン」等)に留める。
- **A-4**: MVPは「bassline×1 + ドラム×1 + 16ステップ再生 + 保存/読み込み + 曲データのJSONエクスポート・インポート(コピペ必須、ファイルは任意)+ 機能優先UI」。フル機能(bassline×2、analog+digital、パターンチェーン/ソング、エフェクト、モダンUI、YAML等の追加フォーマット)は後続の拡張。

## Out of Scope (MVP)
- basslineの2台目、ドラムマシンの2台目(analogとdigitalの両方)。
- エフェクト(Distortion / Delay / PCF / Compressor)。
- パターンチェーン/ソングモード(初期段階では基本パターン再生を優先可)。
- モダンなスキン/装飾UI。
- MIDI入出力、オーディオ書き出し(WAV export)、DAW連携。

## Key Requirements Summary
1. ブラウザ(TypeScript + Web Audio API)で動く、静的配信可能なグルーヴボックス。
2. MVP: basslineスタイル・ベースライン×1 と ドラムマシン×1 を**モデリング合成**で実装。
3. 16ステップ・シーケンサー(look-aheadスケジューリング)。まず基本再生、最終的にアクセント/スライド/シャッフル/ソングへ。
4. パターン/曲の保存・読み込み、および **JSON でのエクスポート(ダウンロード)/インポート(コピペ必須・ファイル任意)**(ラウンドトリップ保証、インポート時は検証)。
5. UIは機能優先で開始し、将来オリジナルのモダンUIへ拡張可能な疎結合設計。
6. 名称・音源・グラフィックはすべて独自(IPはオリジナル)。
7. Security Baseline(Full)適用 — 対象は主に web/クライアント関連ルール。Resiliency は無効。PBT は部分適用。
